# SFT-FFG100 通信调试工具 — 项目介绍

## 概述

本项目是面向 **SFT-FFG100** 设备的 **Windows 桌面调试工具**（协议版本 **V1.3**），基于 **Python 3** 与 **PyQt5** 实现。主要用于通过 **RS485 串口** 完成设备指令调试、运行状态查看、人员信息下发以及（独立高速总线上的）图片传输。

## 目标用户与场景

- 产线或现场工程人员：快速验证设备心跳、复位、开锁逻辑。
- 协议/嵌入开发：对照上位机报文与下位机实现，排查通信问题。
- 需要批量扫描 1～N 号站点在线状态与关键遥测量时，使用批量轮询与设备列表。

## 技术栈

| 类别 | 技术 |
|------|------|
| 语言 | Python 3 |
| 界面 | PyQt5 |
| 串口 | pyserial |
| 可选 | Pillow（汉字 16×16 取模、图片缩放宽高并转 RGB565） |

## 双总线设计

| 总线 | 建议波特率 | 用途 |
|------|------------|------|
| **总线 A** | 默认 **9600**（界面可选 115200） | 指令与状态：心跳、复位、开锁、人员信息帧、批量轮询 |
| **总线 B** | 固定 **115200** | 大图数据分包传输（功能码 0x30 / 0x31 / 0x32） |

总线 A 与总线 B 在软件中对应 **两个独立串口**，需分别打开；物理上可能是两根 USB–串口线或 RS485 两路端口。

## 协议要点（V1.3）

- **无单独帧尾**，依靠 **帧头 `0xAA 0x55`、数据长度字段、CRC16** 完成帧定界与校验。
- **CRC16**：Modbus/RTU 常用多项式（代码中初值 `0xFFFF`，多项式 `0xA001`，低字节在前输出）。
- **总线 A 帧**：长度字段 **1 字节**（数据域长度 N）。
- **总线 B 帧**：长度字段 **2 字节小端**（数据域长度 N），以支持较大图片分包。

应答侧约定：成功类应答功能码多为 **`0x80`**（ACK）；具体解析逻辑见《功能详述》。

## 运行方式

```bash
pip install -r requirements.txt
# 若使用人员信息或图片下发，另需: pip install Pillow
python master_gui.py
```

详细操作步骤、各按钮对应的报文与流程见 **《功能详述》**（`docs/功能详述.md`）。

## 测试

```bash
pytest test_protocol.py
```
